
```bash
# 构建相应镜像
./scripts/build_image.sh docker/lab.env

# 启动相应容器
./scripts/start_container.sh docker/lab.env

# 导出相应的镜像（默认使用 env 文件里的 DOCKER_EXPORT_DIR）
./scripts/export_image.sh docker/lab.env

# 如果需要，也可以手动指定导出路径
./scripts/export_image.sh docker/lab.env /data/E/data_ubuntu/object-rel-nav-cu118.tar

# 删除相关的镜像和容器
./scripts/clean-compose.sh docker/lab.env
```

`./scripts/build_image.sh docker/lab.env` 会优先检查 `DOCKER_IMPORT_DIR` 指定的 tar 路径。
如果该 tar 存在，就直接 `docker load`；否则再执行 compose build。

`./scripts/export_image.sh docker/lab.env` 会导出到 `DOCKER_EXPORT_DIR` 指定的 tar 路径。
